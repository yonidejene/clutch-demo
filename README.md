## Getting Started

1. Install [Expo Go](https://expo.dev/go) on your device or set up the [iOS Simulator](https://docs.expo.dev/get-started/set-up-your-environment/)
2. Run:

```bash
pnpm install
pnpm start
```

3. Scan the QR code with Expo Go or press `i` to open in the Simulator

## Technical Decisions

- **Expo Go (SDK 54) + TypeScript** for the fastest path to a working iOS app without native build config
- **[HeroUI Native](https://v3.heroui.com/docs/native/getting-started) + [Uniwind](https://uniwind.dev) (Tailwind v4)** because I wanted to experiment with newer RN styling tools. Still in beta but the Tailwind-in-RN developer experience was productive
- **[Supabase](https://supabase.com)** for auth, Postgres, and row-level security in one service. Faster to ship than a custom backend
- **[TanStack Query](https://tanstack.com/query)** for infinite scroll pagination with granular cache keys per video, so like/comment mutations only invalidate the affected video instead of the whole feed
- **One video at a time.** Only the active card plays via [`expo-video`](https://docs.expo.dev/versions/latest/sdk/video/); others show thumbnails to keep memory reasonable
- **Optimistic UI** on likes and comments. Cache updates instantly, rolls back on error
- **Custom hooks per feature** (`useFeed`, `useLikes`, `useComments`, `useProfile`) to keep screens thin and logic testable

## Development Approach

I built the product in the following milestones, using Claude Code with a code review from another frontend engineer.

1. **Scaffold:** Expo Router, Supabase client, Uniwind, shared components and theme
2. **Auth:** auto-signup login with carousel, session persistence
3. **Video playback:** one video plays at a time, with mute and play/pause overlay
4. **Feed + social:** infinite scroll, video cards with likes and comments (optimistic UI)
5. **Comments:** dedicated screen with post, delete, and quick-reply chips
6. **Match detail:** full-match video screen with per-card orientation toggle
7. **Profile:** view/edit screen with logout

## Trade-offs & Known Limitations

- Venue names and timestamps are mocked because the API doesn't provide them, so they're generated deterministically from the video ID
- Didn't test on Android. Should work via Expo Go but iOS Simulator was the primary target
- No unit tests yet. I prioritized shipping features; hooks are designed to be testable
- Failed mutations roll back silently with no user-facing feedback (no toasts or error banners)
- Email confirmation disabled in Supabase for demo purposes

## What I'd Do Next

1. **Testing:** unit tests for hooks (especially optimistic rollback logic), integration tests for auth flow
2. **Error UX:** toast notifications for failed mutations instead of silent rollbacks
3. **Video player:** better performance and error handling
4. **Android:** verify layout and video playback, fix platform-specific issues
5. **Better auth.** I used autogenerated usernames + email/password for speed instead of social auth/passwordless
6. **Sharing:** deep links to specific videos, share sheet integration
7. **Polish:** custom app icon, splash screen, loading transitions
